# 第五章 性能优化与调试

## 5.1 性能分析工具详解

### 5.1.1 Stat Unit 统计系统

Stat Unit 是虚幻引擎内置的性能统计工具，专门用于监控动画系统的性能表现。

```cpp
// 启用动画性能统计
STAT(STAT_AnimGameThreadTime);
STAT(STAT_AnimTickTime);
STAT(STAT_SkeletalMeshUpdateTime);

// 在控制台查看统计信息
stat unit
stat anim
stat skeletalmesh
```

**关键性能指标：**
- **Frame Time**：总帧时间，目标60FPS时应在16.67ms以内
- **Game Thread Time**：游戏线程时间，动画计算主要在此线程
- **GPU Time**：GPU渲染时间，骨骼蒙皮计算开销

### 5.1.2 Animation Insights 深度分析

Animation Insights 是专门针对动画系统的性能分析工具，提供详细的调用堆栈和耗时分析。

**使用步骤：**
1. 打开 Window → Developer Tools → Animation Insights
2. 开始录制性能数据
3. 运行游戏场景
4. 停止录制并分析数据

**分析重点：**
- **Anim Tick**：动画更新耗时
- **Skeletal Mesh Update**：骨骼网格更新耗时
- **Montage Playback**：蒙太奇播放性能
- **Blend Tree Evaluation**：混合树计算开销

### 5.1.3 GPU Profiler 分析

GPU Profiler 用于分析骨骼蒙皮在GPU上的性能表现。

```cpp
// GPU性能标记
BEGIN_DRAW_EVENT(GPU, SkeletalMeshRendering);
// 骨骼蒙皮渲染代码
END_DRAW_EVENT(GPU);
```

**GPU性能优化点：**
- **顶点着色器复杂度**：骨骼数量对顶点着色器的影响
- **骨骼矩阵上传**：CPU到GPU的数据传输开销
- **蒙皮缓存**：骨骼变换的缓存机制

## 5.2 内存优化策略

### 5.2.1 动画压缩技术

虚幻引擎支持多种动画压缩算法，选择合适的压缩方式对内存优化至关重要。

**压缩算法比较：**

| 算法类型 | 压缩比 | 质量损失 | 适用场景 |
|---------|--------|----------|----------|
| **ACL** | 高 | 低 | 高质量角色动画 |
| **Bitwise** | 中 | 中 | 通用动画数据 |
| **Adaptive** | 可变 | 可变 | 动态质量调整 |
| **Lossy** | 很高 | 高 | 低优先级动画 |

**压缩设置示例：**
```cpp
// 在动画资源中设置压缩
UAnimSequence* AnimSequence;
AnimSequence->CompressionScheme = NewObject<UAnimCompress>();
AnimSequence->bEnableCompression = true;
AnimSequence->CompressionErrorThreshold = 0.1f; // 误差阈值
```

### 5.2.2 流式加载策略

对于大型动画项目，流式加载可以显著减少内存占用。

**异步加载实现：**
```cpp
// 异步加载动画资源
FStreamableManager StreamableManager;
TSharedPtr<FStreamableHandle> Handle = StreamableManager.RequestAsyncLoad(
    AnimAssetPath,
    FStreamableDelegate::CreateUObject(this, &ACharacter::OnAnimLoaded)
);

void ACharacter::OnAnimLoaded()
{
    // 动画资源加载完成后的处理
    USkeletalMeshComponent* SkeletalMesh = GetMesh();
    SkeletalMesh->SetAnimInstanceClass(LoadedAnimClass);
}
```

**内存管理策略：**
- **按需加载**：只在需要时加载动画资源
- **LRU缓存**：最近最少使用算法的缓存管理
- **内存池**：预分配内存池减少分配开销

### 5.2.3 LOD系统优化

多层次细节（LOD）系统可以显著降低动画计算开销。

**骨骼LOD设置：**
```cpp
// 骨骼LOD配置
USkeletalMesh* SkeletalMesh;
SkeletalMesh->LODInfo.SetNum(4); // 设置4个LOD级别

// LOD 0：全精度骨骼（近距离）
SkeletalMesh->LODInfo[0].LODHysteresis = 0.02f;
SkeletalMesh->LODInfo[0].ScreenSize = 1.0f;

// LOD 1：中等精度骨骼（中距离）
SkeletalMesh->LODInfo[1].LODHysteresis = 0.01f;
SkeletalMesh->LODInfo[1].ScreenSize = 0.5f;

// LOD 2：低精度骨骼（远距离）
SkeletalMesh->LODInfo[2].LODHysteresis = 0.005f;
SkeletalMesh->LODInfo[2].ScreenSize = 0.2f;
```

**动画LOD策略：**
- **简化骨骼数量**：远距离减少活动骨骼数
- **降低更新频率**：远距离降低动画更新频率
- **简化混合计算**：减少复杂的混合计算

## 5.3 CPU优化技巧

### 5.3.1 并行更新优化

**并行动画更新：**

```cpp
// 线程安全的动画更新
virtual void NativeThreadSafeUpdateAnimation(float DeltaSeconds) override
{
    // 在此函数中进行线程安全的动画计算
    // 避免访问非线程安全的游戏对象
    
    // 安全的变量访问
    float SafeSpeed = GetThreadSafeSpeed();
    
    // 避免直接访问Actor或Component
    // ❌ GetOwner()->GetVelocity() - 不安全

```

### 5.3.2 缓存与重用策略

通过缓存机制避免重复计算是重要的优化手段。

**骨骼变换缓存：**
```cpp
// 骨骼变换缓存实现
struct FBoneTransformCache
{
    TArray<FTransform> CachedTransforms;
    bool bIsDirty;
    
    void UpdateCache(const TArray<FTransform>& NewTransforms)
    {
        if (bIsDirty || CachedTransforms.Num() != NewTransforms.Num())
        {
            CachedTransforms = NewTransforms;
            bIsDirty = false;
        }
    }
    
    const TArray<FTransform>& GetCachedTransforms() const
    {
        return CachedTransforms;
    }
};
```

**动画数据缓存：**
- **姿势缓存**：缓存常用姿势避免重复计算
- **混合结果缓存**：缓存混合计算结果
- **IK解算缓存**：缓存IK计算结果

### 5.3.3 提前剔除优化

通过可见性判断提前剔除不可见物体的动画计算。

**视锥体剔除：**
```cpp
// 视锥体剔除检查
bool IsInViewFrustum(const FVector& Location, float BoundingRadius)
{
    if (GEngine && GEngine->GameViewport)
    {
        FSceneViewFamilyContext ViewFamily;
        FSceneView* View = GEngine->GameViewport->CreateSceneView(&ViewFamily);
        
        return View->ViewFrustum.IntersectSphere(Location, BoundingRadius);
    }
    return true;
}

// 在动画更新前进行检查
void UpdateAnimationIfVisible()
{
    if (IsInViewFrustum(GetActorLocation(), BoundingRadius))
    {
        UpdateAnimation();
    }
    else
    {
        // 不可见时跳过动画更新
        SkipAnimationUpdate();
    }
}
```

**距离剔除策略：**
- **近场动画**：全精度更新（<50米）
- **中场动画**：简化更新（50-200米）
- **远场动画**：最低精度或跳过（>200米）

### 5.4.3 性能基准测试

建立性能基准可以帮助量化优化效果。

**基准测试框架：**
```cpp
// 性能基准测试类
class FAnimationBenchmark
{
public:
    void RunBenchmark()
    {
        // 测试不同骨骼数量的性能
        TestBoneCountPerformance();
        
        // 测试动画混合性能
        TestBlendingPerformance();
        
        // 测试IK计算性能
        TestIKPerformance();
    }
    
private:
    void TestBoneCountPerformance()
    {
        TArray<int32> BoneCounts = {50, 100, 200, 500};
        
        for (int32 BoneCount : BoneCounts)
        {
            FDateTime StartTime = FDateTime::Now();
            
            // 执行测试
            SimulateAnimationUpdate(BoneCount);
            
            FDateTime EndTime = FDateTime::Now();
            FTimespan Duration = EndTime - StartTime;
            
            UE_LOG(LogAnimation, Log, TEXT("BoneCount: %d, Time: %.3f ms"), 
                BoneCount, Duration.GetTotalMilliseconds());
        }
    }
};
```

## 5.5 实战优化案例

### 5.5.1 大规模角色场景优化

**问题场景：** 100个角色同时播放复杂动画

**优化方案：**
1. **LOD分级**：根据距离使用不同精度的动画
2. **更新频率控制**：远距离角色降低更新频率
3. **骨骼简化**：减少不重要的骨骼计算
4. **实例化渲染**：相同角色的动画实例化

### 5.5.2 复杂混合树优化

**问题场景：** 多层混合树导致CPU开销过高

**优化方案：**
1. **缓存混合结果**：缓存常用混合组合
2. **简化混合层级**：减少不必要的混合层
3. **异步计算**：非关键混合异步处理
4. **预计算优化**：离线计算部分混合结果

### 5.5.3 移动平台优化

**问题场景：** 移动设备性能受限

**优化方案：**
1. **降低骨骼精度**：减少骨骼数量
2. **简化着色器**：使用移动端优化着色器
3. **纹理压缩**：使用移动端纹理格式
4. **内存优化**：严格控制内存使用
