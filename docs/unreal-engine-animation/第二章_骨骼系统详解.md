# 第二章：骨骼系统详解

## 2.1 SkeletalMesh组件架构

### 核心组件结构
```cpp
// SkeletalMeshComponent 核心属性
class USkeletalMeshComponent : public USkinnedMeshComponent {
public:
    // 骨骼网格资源
    USkeletalMesh* SkeletalMesh;
    
    // 动画实例
    UAnimInstance* AnimInstance;
    
    // 骨骼变换数据
    TArray<FTransform> ComponentSpaceTransforms;
    TArray<FTransform> LocalSpaceTransforms;
};
```

### 组件功能模块
- **渲染模块**：骨骼网格的绘制和材质管理
- **动画模块**：动画播放和混合计算
- **物理模块**：碰撞检测和物理模拟
- **事件模块**：动画通知和回调处理

## 2.2 骨骼变换计算原理

### 骨骼层级关系
```
Root (根骨骼)
├── Pelvis (骨盆)
│   ├── Spine_01 (脊柱1)
│   │   ├── Spine_02 (脊柱2)
│   │   │   ├── Neck (颈部)
│   │   │   │   └── Head (头部)
│   │   │   ├── Clavicle_L (左锁骨)
│   │   │   │   └── UpperArm_L (左上臂)
│   │   │   └── Clavicle_R (右锁骨)
│   │   │       └── UpperArm_R (右上臂)
│   │   └── Thigh_L (左大腿)
│   │       └── Calf_L (左小腿)
│   └── Thigh_R (右大腿)
│       └── Calf_R (右小腿)
└── 其他骨骼...
```

### 变换计算流程
1. **局部空间变换**：骨骼相对于父骨骼的变换
2. **世界空间变换**：骨骼在场景中的最终位置
3. **绑定姿势矩阵**：骨骼的初始参考姿势
4. **逆绑定姿势矩阵**：用于蒙皮计算

### 矩阵计算公式
```cpp
// 局部到组件空间变换
FTransform LocalToComponent = ParentTransform * LocalTransform;

// 组件到世界空间变换  
FTransform ComponentToWorld = GetComponentTransform();

// 最终骨骼变换
FTransform BoneTransform = ComponentToWorld * LocalToComponent;
```

## 2.3 骨骼修改与扩展技术

### 骨骼控制器（Bone Controller）
**功能**：实时修改骨骼变换
**应用场景**：
- 动态调整角色姿势
- 物理驱动的骨骼动画
- 程序化动画生成

### 物理骨骼系统
**核心组件**：
- **PhysicsAsset**：物理资产定义
- **PhysicsConstraint**：物理约束
- **Ragdoll**：布娃娃物理系统

### 骨骼插槽（Sockets）
**功能**：在骨骼上附加物体
**配置参数**：
- 相对位置和旋转
- 缩放比例
- 插槽名称和类型

## 2.4 性能优化策略

### 骨骼LOD系统
**多级细节**：
- LOD0：全精度骨骼（近距离）
- LOD1：简化骨骼（中距离）
- LOD2：极简骨骼（远距离）

### 骨骼剔除优化
**不可见骨骼跳过计算**：
```cpp
// 检查骨骼可见性
if (!BoneIsVisible(BoneIndex)) {
    // 跳过变换计算
    continue;
}
```

### 并行计算优化
**多线程骨骼更新**：
```cpp
// 并行处理骨骼变换
ParallelFor(NumBones, [&](int32 BoneIndex) {
    UpdateBoneTransform(BoneIndex);
});
```

## 2.5 调试与问题排查

### 常见问题
1. **骨骼错位**：变换计算错误
2. **蒙皮撕裂**：权重分配问题
3. **性能瓶颈**：骨骼数量过多
