# 第一章：基础概念与架构

## 1.1 虚幻引擎动画系统概述

### 系统架构总览

虚幻引擎（Unreal Engine，以下简称 UE）的动画系统是业界最成熟的实时动画解决方案之一，支持从手机游戏到 AAA 主机大作的全平台开发。其核心设计哲学是**高度模块化、可扩展、数据驱动**。

```
┌─────────────────────────────────────────────────────────────┐
│                   UE5 动画系统架构总览                        │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │  数据层       │  │  逻辑层       │  │    渲染层        │  │
│  │              │  │              │  │                  │  │
│  │ Skeleton     │  │ AnimInstance  │  │ SkeletalMesh     │  │
│  │ AnimSequence │──▶│ AnimBP       │──▶│ Component       │  │
│  │ AnimMontage  │  │ State Machine │  │ Skinning         │  │
│  │ BlendSpace   │  │ AnimGraph     │  │ LOD              │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
│                           │                                  │
│                    ┌──────▼──────┐                           │
│                    │  Physics     │                           │
│                    │  Integration │                           │
│                    │ (Chaos/PhAT) │                           │
│                    └─────────────┘                           │
└─────────────────────────────────────────────────────────────┘
```

### 核心理念：分层解耦

UE 动画系统将整个管线分为以下几个清晰的层次：

| 层次 | 职责 | 核心类 |
|------|------|--------|
| **数据层** | 存储原始动画数据 | `UAnimSequence`, `USkeleton` |
| **评估层** | 计算每帧的骨骼变换 | `FAnimNode_Base`, `FPoseContext` |
| **逻辑层** | 控制动画状态与混合 | `UAnimInstance`, `UAnimBlueprintGeneratedClass` |
| **渲染层** | 将骨骼变换应用到网格 | `USkeletalMeshComponent`, `FSkeletalMeshObject` |
| **物理层** | 物理驱动的动画修正 | `UPhysicsAsset`, `FAnimNode_RigidBody` |

### UE4 vs UE5 核心差异

| 特性 | UE4 | UE5 |
|------|-----|-----|
| 默认骨骼网格体 | Quinn/Mannequin | Manny/Quinn (重新设计) |
| IK 系统 | FABRIK | Full Body IK (FBIK) |
| 动画重定向 | 基于骨骼比例 | IK Retargeter（更精确）|
| Control Rig | 实验性 | 正式集成，功能完整 |
| 布料模拟 | Apex Cloth | Chaos Cloth |
| 物理引擎 | PhysX | Chaos Physics |
| Procedural Animation | 有限 | Motion Warping, Pose Warping 等 |

## 1.2 骨骼系统基础

### 什么是骨骼（Skeleton）

在 UE 中，**Skeleton（骨骼资产）** 是一个独立的资产文件（`.uasset`），它定义了骨骼的层级结构（Bone Hierarchy）。多个 Skeletal Mesh 可以共享同一个 Skeleton，从而共享动画数据。

```
根骨骼 (Root)
├── 盆骨 (Pelvis)
│   ├── 脊椎1 (Spine_01)
│   │   ├── 脊椎2 (Spine_02)
│   │   │   ├── 脊椎3 (Spine_03)
│   │   │   │   ├── 颈部 (Neck)
│   │   │   │   │   └── 头部 (Head)
│   │   │   │   ├── 左锁骨 (Clavicle_L)
│   │   │   │   │   └── 左上臂 → 左前臂 → 左手
│   │   │   │   └── 右锁骨 (Clavicle_R)
│   │   │   │       └── 右上臂 → 右前臂 → 右手
│   ├── 左大腿 (Thigh_L)
│   │   └── 左小腿 → 左脚 → 左脚趾
│   └── 右大腿 (Thigh_R)
│       └── 右小腿 → 右脚 → 右脚趾
└── IK骨骼 (IK_Foot_Root 等)
```

### Skeleton 与 Skeletal Mesh 的关系

```
USkeleton（骨骼资产）
    ├── 定义骨骼名称和层级
    ├── 存储 Reference Pose（T-Pose/A-Pose）
    ├── 包含 Socket 定义
    └── 关联多个 Skeletal Mesh
         ├── Mesh_Char_A.uasset（共享骨骼）
         ├── Mesh_Char_B.uasset（共享骨骼）
         └── Mesh_Char_C.uasset（共享骨骼）
```

**关键点**：只要骨骼名称和层级相同，不同形状的 Mesh 都可以共享同一套动画——这是 UE 动画系统的强大之处。

### 骨骼空间（Bone Space）说明

在处理骨骼变换时，有三种空间需要区分：

| 空间名称 | 说明 | 使用场景 |
|---------|------|--------|
| **Local Space（局部空间）** | 相对于父骨骼的变换 | 动画数据存储格式 |
| **Component Space（组件空间）** | 相对于 SkeletalMeshComponent 根部 | IK 计算、骨骼控制节点 |
| **World Space（世界空间）** | 相对于世界原点 | 物理交互、武器挂载点 |

> 📸 **图示说明**：在 UE 编辑器 → Skeleton Editor 中，可以通过 "Local/Component" 切换按钮查看不同空间下的骨骼变换预览。

## 1.3 动画资源类型

### Animation Sequence（动画序列）
最基础的动画资产，记录了骨骼在时间轴上的完整变换数据。

- **格式**：每帧存储所有骨骼的 Position/Rotation/Scale
- **压缩**：支持多种压缩算法（ACL 为 UE5 默认）
- **典型用途**：行走、跑步、攻击等基础动作

### Animation Montage（动画蒙太奇）
在 Animation Sequence 基础上添加了分段（Section）和槽（Slot）的高级资产。

```
Animation Montage 结构：
┌─────────────────────────────────────────────┐
│  Slot: UpperBody                             │
│  ┌──────────────────────────────────────┐   │
│  │ Section: Attack  │ Section: End      │   │
│  │ [攻击动画]        │ [收招动画]         │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  Notifies Track:                             │
│  ──●────────────●────────────────────────   │
│    HitDetect    FootStep                    │
└─────────────────────────────────────────────┘
```

**典型应用**：攻击动作、受击、翻滚等需要程序控制播放的动画。

### Blend Space（混合空间）
根据一个或两个输入参数（如速度、方向）在多个动画之间平滑插值。

```
2D Blend Space 示意：
                    前(180)
                      ↑
  左跑(L-Run)  ←──────┼──────→  右跑(R-Run)
  左走(L-Walk)         ↓         右走(R-Walk)
                   后(0/-180)
          Y轴: 速度 (0→600)
          X轴: 方向角 (-180→180)
```

### Pose Asset（姿势资产）
存储骨骼的静态姿势，常用于面部动画的 Morph Target 替代方案，或与 Pose Blender 节点配合使用。

### Aim Offset（瞄准偏移）
一种特殊的 Blend Space，专门用于上半身瞄准方向的叠加动画，接受 Pitch/Yaw 作为输入。

## 1.4 动画管线架构

### 从美术到游戏的完整流程

```
┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│  DCC 软件    │     │  UE 导入     │     │  运行时评估      │
│  (Maya/Max/ │────▶│  FBX Import  │────▶│                 │
│   Blender)  │     │  • Skeleton  │     │  AnimInstance   │
│             │     │  • Mesh      │     │  ──▶ AnimGraph  │
│  输出:       │     │  • AnimSeq   │     │  ──▶ PoseEval  │
│  • FBX      │     │  • Materials │     │  ──▶ Skinning  │
│  • Alembic  │     └──────────────┘     │  ──▶ Render    │
│  • LiveLink │                          └─────────────────┘
└─────────────┘
```

### 运行时动画评估管线

每一帧，UE 按以下顺序处理动画：

1. **Tick Animation** (`USkeletalMeshComponent::TickAnimation`)
   - 更新动画时间、触发 Notify
2. **Evaluate Animation** (`USkeletalMeshComponent::EvaluateAnimation`)
   - 执行 AnimGraph，生成 Local Space Pose
3. **Fill SpaceBases** (`USkeletalMeshComponent::FillComponentSpaceTransforms`)
   - 将 Local Pose 转换为 Component Space
4. **Update Cloth / Physics** (物理后处理)
5. **Finalize** → 提交给 GPU Skinning

## 1.5 核心组件详解

### USkeletalMeshComponent

这是驱动整个角色动画的核心组件，继承自 `UMeshComponent`。

```cpp
// 获取 SkeletalMeshComponent 并设置动画
UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Components")
USkeletalMeshComponent* SkeletalMesh;

// 在 BeginPlay 中初始化
void AMyCharacter::BeginPlay()
{
    Super::BeginPlay();
    
    // 获取组件
    SkeletalMesh = GetMesh();
    
    // 动态设置动画蓝图类
    if (AnimBlueprintClass)
    {
        SkeletalMesh->SetAnimInstanceClass(AnimBlueprintClass);
    }
    
    // 获取 AnimInstance 实例
    UAnimInstance* AnimInst = SkeletalMesh->GetAnimInstance();
    if (UMyAnimInstance* MyAnimInst = Cast<UMyAnimInstance>(AnimInst))
    {
        MyAnimInst->OnMontageEnded.AddDynamic(this, &AMyCharacter::OnMontageEnded);
    }
}
```

### 重要属性

| 属性 | 说明 |
|------|------|
| `AnimationMode` | `AnimationBlueprint` 或 `SingleAnimation` |
| `AnimClass` | 关联的动画蓝图类 |
| `bNoSkeletonUpdate` | 是否跳过骨骼更新（用于优化不可见角色）|
| `VisibilityBasedAnimTickOption` | 基于可见性的 Tick 策略 |
| `ClothingSimulationInteractor` | 布料仿真交互器 |

### Skeletal Mesh 的 LOD 系统

LOD（Level of Detail，细节层次）是 UE 动画系统中最重要的性能手段之一。

```
LOD0: 角色近处，完整骨骼（50根骨骼，5000三角面）
LOD1: 中等距离，减少骨骼（30根骨骼，2500三角面）
LOD2: 较远距离，简化骨骼（15根骨骼，1000三角面）
LOD3: 最远距离，最少骨骼（5根骨骼，400三角面）
```

```cpp
// 设置 LOD 偏移（强制使用特定 LOD）
SkeletalMeshComp->SetForcedLOD(2); // 强制使用 LOD2

// 动态调整 LOD 距离
SkeletalMeshComp->SetCachedMaxDrawDistance(1000.0f);

// 禁用特定 LOD 的动画更新
SkeletalMeshComp->SetComponentTickEnabled(false); // 谨慎使用
```

## 2.2 骨骼变换计算

### 变换的数学基础

UE 使用**四元数（Quaternion）**表示骨骼旋转，而非欧拉角（Euler Angles）。这避免了万向节锁（Gimbal Lock）问题。

```cpp
// FTransform 结构（位置 + 旋转 + 缩放）
FTransform BoneTransform;
BoneTransform.SetLocation(FVector(10.0f, 0.0f, 50.0f));
BoneTransform.SetRotation(FQuat(FRotator(0.0f, 90.0f, 0.0f)));
BoneTransform.SetScale3D(FVector(1.0f, 1.0f, 1.0f));

// 从四元数创建旋转
FQuat Rotation = FQuat(FVector(0, 0, 1), FMath::DegreesToRadians(90.0f));

// 四元数插值（动画混合的核心）
FQuat BlendedRot = FQuat::Slerp(RotA, RotB, Alpha); // 球形线性插值
FQuat BlendedRot2 = FQuat::FastLerp(RotA, RotB, Alpha); // 快速但不完全精确

// 骨骼变换的级联（子骨骼的世界变换 = 父变换 × 子本地变换）
FTransform ParentWorld = GetBoneTransform(ParentBoneIdx);
FTransform ChildLocal = GetLocalTransform(ChildBoneIdx);
FTransform ChildWorld = ChildLocal * ParentWorld;
```

### 蒙皮（Skinning）算法

线性混合蒙皮（LBS，Linear Blend Skinning）是 UE 的默认蒙皮方式：

```
顶点最终位置 = Σ (权重[i] × 骨骼矩阵[i] × 顶点初始位置)

例如，肩膀处的顶点可能受两根骨骼影响：
- 上臂骨骼，权重 0.6
- 脊椎骨骼，权重 0.4
最终位置 = 0.6 × UpperArm_Matrix × V + 0.4 × Spine_Matrix × V
```

UE5 支持 **DQS（Dual Quaternion Skinning）**，可有效解决 LBS 在大角度旋转时的"糖果纸扭曲"问题：

```cpp
// 在 Skeletal Mesh 编辑器 → Details → Skin Weights 下
// 设置 Skin Weight Profile 并选择 Dual Quaternion 模式
// 或在 C++ 中：
SkeletalMeshComp->SkinWeightProfileName = FName("DualQuaternion");
```

## 2.4 骨骼层级与权重

### 蒙皮权重编辑

在 UE 编辑器中，**Skin Weight Paint** 工具允许美术在引擎内直接调整蒙皮权重，无需返回 DCC 软件。

权重调整原则：
- 关节中心：靠近关节的骨骼权重应接近 1.0
- 过渡区域：多骨骼平滑混合，避免硬切
- 面部/手指：精细权重，建议使用 DCC 软件完成后导入

### Bone Chain（骨骼链）

IK 系统依赖骨骼链的正确定义：

```
脊椎链：Spine_01 → Spine_02 → Spine_03（3骨骼链）
手臂链：UpperArm → LowerArm → Hand（3骨骼链）
腿部链：Thigh → Calf → Foot（3骨骼链）
```

## 2.5 骨骼动画数学基础

### 矩阵变换流水线

```
世界变换矩阵 = 平移矩阵 × 旋转矩阵 × 缩放矩阵

骨骼层级传递：
Root_World = Root_Local
Spine_World = Spine_Local × Root_World
Head_World  = Head_Local  × Spine_World

蒙皮矩阵（最终用于 GPU）：
SkinMatrix[i] = BoneWorld[i] × InverseBind[i]
其中 InverseBind = 参考姿势下骨骼世界变换的逆矩阵
```

```cpp
// UE 中获取骨骼的蒙皮矩阵（内部实现参考）
const TArray<FMatrix>& RefToLocal = SkeletalMeshComp->GetBoneMatrices();
// RefToLocal[BoneIdx] 就是第 BoneIdx 根骨骼的最终蒙皮矩阵
```

### 插值方法对比

| 方法 | 精度 | 性能 | 使用场景 |
|------|------|------|--------|
| Lerp（线性插值）| 低（旋转不准确）| 最快 | 位置/缩放插值 |
| Slerp（球形线性插值）| 高 | 较慢 | 旋转插值（标准）|
| FastLerp | 中 | 快 | 旋转插值（需要速度时）|
| Nlerp（归一化线性插值）| 较高 | 快 | 旋转插值（近似 Slerp）|

## 3.1 Animation Sequence 详解

### 内部数据结构

每个 `UAnimSequence` 内部存储的核心数据：

```cpp
// 简化的 AnimSequence 内部结构（参考源码）
class UAnimSequence : public UAnimSequenceBase
{
public:
    // 动画总时长（秒）
    float SequenceLength;
    
    // 帧率
    float RateScale = 1.0f;
    
    // 压缩后的骨骼轨道数据
    TArray<FCompressedSegment> CompressedSegments;
    
    // 曲线数据（Float Curves, Transform Curves）
    FRawCurveTracks RawCurveData;
    
    // Root Motion 数据
    FRootMotionData RootMotionData;
    
    // 动画通知
    TArray<FAnimNotifyEvent> Notifies;
};
```

### Root Motion（根运动）

Root Motion 允许动画数据直接驱动角色移动，而非由代码控制位移。

```
普通动画：角色位置由 CharacterMovement 控制，动画只是"播放"
Root Motion：动画的根骨骼位移被提取出来，转化为实际的角色移动

配置步骤：
1. AnimSequence → Details → Root Motion → Enable Root Motion ✅
2. AnimBP → Root Motion Mode = Root Motion from Everything (推荐)
   或 Root Motion from Montages Only (仅蒙太奇有效)
```

```cpp
// 在 Character 中配置 Root Motion
void AMyCharacter::SetupRootMotion()
{
    // 获取角色移动组件
    UCharacterMovementComponent* MovComp = GetCharacterMovement();
    
    // 允许根运动覆盖移动
    MovComp->bAllowPhysicsRotationDuringAnimRootMotion = true;
}

// 在 AnimInstance 中检测根运动
void UMyAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    Super::NativeUpdateAnimation(DeltaSeconds);
    
    // 获取根运动的增量变换
    FRootMotionMovementParams RMParams = GetProxyOnGameThread<FAnimInstanceProxy>()
        .ConsumeExtractedRootMotion(1.0f);
    
    // RMParams.bHasRootMotion 判断是否有根运动
}
```

## 3.2 动画混合技术

### 基础混合（Blend）

动画混合是动画系统的核心能力，允许在多个动画之间平滑过渡：

```
混合公式：
Pose_Result = Pose_A × (1 - Alpha) + Pose_B × Alpha

当 Alpha = 0：完全播放 Pose_A
当 Alpha = 0.5：两者各占 50%
当 Alpha = 1：完全播放 Pose_B
```

### 分层混合（Layered Blend）

分层混合允许上下半身使用不同动画：

```cpp
// AnimGraph 中的 Layered Blend Per Bone 节点配置
// Base Pose: 完整的行走动画
// Blend Pose 0: 射击上半身动画
// Layer Setup:
//   Branch Filter: spine_01（从脊椎向上的所有骨骼受 Blend Pose 影响）
//   Blend Depth: -1（-1 表示完全覆盖分支骨骼，0 表示混合）
```

蓝图实现逻辑（伪代码）：
```
[Locomotion SM] ──▶ [Layered Blend Per Bone] ──▶ [Output Pose]
                              ↑
[Shooting Upper Body] ────────┘
（Alpha = bIsAiming ? 1.0 : 0.0）
```

### Blend Space 采样实现

```cpp

void UMyAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    Super::NativeUpdateAnimation(DeltaSeconds);
    
    APawn* Pawn = TryGetPawnOwner();
    if (!Pawn) return;
    
    // 计算速度（用于混合空间 Y 轴）
    FVector Velocity = Pawn->GetVelocity();
    Speed = Velocity.Size();
    
    // 计算移动方向（用于混合空间 X 轴）
    if (Speed > 0.1f)
    {
        FRotator ActorRot = Pawn->GetActorRotation();
        FVector VelocityDir = Velocity.GetSafeNormal();
        
        // 将速度方向转换为相对于角色朝向的角度
        FVector ForwardVec = ActorRot.Vector();
        FVector RightVec = FRotationMatrix(ActorRot).GetScaledAxis(EAxis::Y);
        
        float ForwardDot = FVector::DotProduct(VelocityDir, ForwardVec);
        float RightDot = FVector::DotProduct(VelocityDir, RightVec);
        Direction = FMath::RadiansToDegrees(FMath::Atan2(RightDot, ForwardDot));
    }
    else
    {
        Direction = 0.0f;
    }
}
```

## 3.4 动画压缩与优化

### UE5 默认压缩方案：ACL

UE5 默认使用 **Animation Compression Library（ACL）** 进行动画数据压缩：

| 压缩方案 | 压缩比 | 精度 | 性能 |
|---------|--------|------|------|
| **ACL（推荐）** | ~10:1 | 高 | 解压极快 |
| Bitwise Compress | ~4:1 | 中 | 快 |
| Remove Linear Keys | ~2:1 | 高 | 快 |
| 无压缩 | 1:1 | 完美 | 最快（内存开销大）|

### 压缩设置

```cpp
// 在 AnimSequence Details → Compression → Compression Settings
// 推荐设置：
// Codec: ACL
// Error Threshold: 0.01 cm（越小越精确，文件越大）
// Rotation Format: QuatDropW（丢弃 W 分量，从 XYZ 重建，节省 25% 空间）
// Translation Format: Float 96（精确）或 Fixed 48（节省内存）
// Scale Format: Float 96（通常可用 Uniform Scale 进一步节省）
```

### 骨骼精简策略

```
优化前：50 骨骼，所有骨骼都被压缩存储
优化后：
  - 移除 LOD1+ 不需要的骨骼（如小骨骼、次要骨骼）
  - 使用 "Remove Trivial Keys" 消除无变化的帧
  - 对远处角色使用 UpdateRate 降低更新频率
```

## 4.2 状态机设计

### 状态机基础概念

```
状态机（State Machine）= 状态（States）+ 转换（Transitions）

典型角色运动状态机：
┌───────────────────────────────────────────────┐
│                  运动状态机                    │
│                                               │
│   ┌────────┐    Speed>10    ┌────────────┐   │
│   │ Idle   │──────────────▶│  Moving    │   │
│   └────────┘               └────────────┘   │
│       ▲     ◀──Speed<5──────     │          │
│       │                          │ Jump      │
│       │                    ┌─────▼──────┐   │
│       └────────────────────│  InAir     │   │
│              Landed        └────────────┘   │
└───────────────────────────────────────────────┘
```

### 转换条件（Transition Rules）

在 UE 状态机中，每条转换箭头都有对应的**转换规则**（Transition Rule），本质是一个返回 bool 的 AnimGraph 节点表达式。

```
Idle → Moving 的转换规则：
  Speed > 10.0 AND NOT bIsInAir
  
  蓝图节点组合：
  [Get Speed] → [>] → [AND] → [Result]
                ↑     ↑
               10   [NOT bIsInAir]
```

### 多层状态机（Hierarchical State Machine）

UE5 支持在状态机内嵌套子状态机（Sub-State Machine），处理复杂逻辑：

```
主状态机
├── Idle
├── Moving（子状态机）
│   ├── Walk
│   ├── Jog
│   └── Sprint
├── InAir
│   ├── Jump
│   ├── DoubleJump
│   └── Fall
└── Actions（子状态机）
    ├── Dodge
    ├── Attack
    └── Death
```

```cpp
USTRUCT(BlueprintInternalUseOnly)
struct FAnimNode_CustomBlend : public FAnimNode_Base
{
    GENERATED_BODY()
    
public:
    // 输入姿势接口
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Input Poses")
    FPoseLink BasePose;
    
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Input Poses")
    FPoseLink AdditivePose;
    
    // 混合权重
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Settings", 
              meta = (PinShownByDefault))
    float Alpha = 1.0f;
    
    // ── FAnimNode_Base 接口 ──
    
    virtual void Initialize_AnyThread(const FAnimationInitializeContext& Context) override;
    virtual void CacheBones_AnyThread(const FAnimationCacheBonesContext& Context) override;
    virtual void Update_AnyThread(const FAnimationUpdateContext& Context) override;
    virtual void Evaluate_AnyThread(FPoseContext& Output) override;
};

// 实现
void FAnimNode_CustomBlend::Evaluate_AnyThread(FPoseContext& Output)
{
    // 评估 Base Pose
    FPoseContext BaseContext(Output);
    BasePose.Evaluate(BaseContext);
    
    // 评估 Additive Pose
    FPoseContext AdditiveContext(Output);
    AdditivePose.Evaluate(AdditiveContext);
    
    // 自定义混合逻辑
    Output.Pose.CopyBonesFrom(BaseContext.Pose);
    
    const int32 NumBones = Output.Pose.GetNumBones();
    for (int32 BoneIdx = 0; BoneIdx < NumBones; ++BoneIdx)
    {
        FCompactPoseBoneIndex CompactBoneIdx(BoneIdx);
        FTransform& BaseTrans = Output.Pose[CompactBoneIdx];
        const FTransform& AddTrans = AdditiveContext.Pose[CompactBoneIdx];
        
        // 这里实现你的自定义混合逻辑
        BaseTrans.BlendWith(AddTrans, Alpha);
    }
}
```

## 4.4 混合空间应用

### 1D Blend Space vs 2D Blend Space

```
1D Blend Space（单轴混合）：
用途：仅根据速度混合行走/跑步
轴：速度 (0 → 600 cm/s)
节点：Idle(0) ─────Walk(200)─────Jog(400)─────Sprint(600)

2D Blend Space（双轴混合）：
用途：同时考虑速度和移动方向（8方向运动）
X轴：方向角 (-180 → 180 度)
Y轴：速度 (0 → 600 cm/s)

     600 [SprintBwd][SprintL][Sprint][SprintR][SprintBwd]
     400 [JogBwd]  [JogL]  [Jog]  [JogR]  [JogBwd]
     200 [WalkBwd] [WalkL] [Walk] [WalkR] [WalkBwd]
       0 [Idle]
      -180  -90     0     90    180
```

### Aim Offset 实现

```cpp
// 在 AnimInstance 中计算瞄准角度
void UMyAnimInstance::UpdateAimOffset()
{
    if (!OwnerCharacter) return;
    
    // 获取角色旋转
    FRotator CharacterRotation = OwnerCharacter->GetActorRotation();
    
    // 获取控制器旋转（瞄准方向）
    FRotator AimRotation = OwnerCharacter->GetBaseAimRotation();
    
    // 计算相对角度
    FRotator DeltaRot = AimRotation - CharacterRotation;
    DeltaRot.Normalize();
    
    // AimYaw: 水平瞄准角（-90 到 90 度，用于 X 轴）
    AimYaw = FMath::ClampAngle(DeltaRot.Yaw, -90.0f, 90.0f);
    
    // AimPitch: 垂直瞄准角（-90 到 90 度，用于 Y 轴）
    AimPitch = FMath::ClampAngle(DeltaRot.Pitch, -90.0f, 90.0f);
}
```

## 6.1 逆向运动学（IK）

### UE5 Full Body IK（FBIK）

UE5 的 Full Body IK 系统基于 **XPBD（Extended Position Based Dynamics）** 求解器，支持多目标、多约束的全身 IK：

```
FBIK 求解链：
目标效应器（End Effectors）：
  • IK_Foot_L → 左脚落点
  • IK_Foot_R → 右脚落点
  • IK_Hand_L → 左手持枪/攀爬点
  • IK_Hand_R → 右手持枪/攀爬点

约束：
  • 骨骼旋转限制（肘关节只能向一个方向弯曲）
  • 位置约束（脚不能穿透地面）
  • 根骨骼修正（腰部随腿部 IK 调整高度）
```

### 脚步 IK 实现（完整示例）

```cpp
// FootIK AnimGraph 节点连接（蓝图伪代码）：
// 1. 每帧用 LineTrace 检测左右脚的地面高度
// 2. 根据地面法线调整脚的旋转
// 3. 用 Two Bone IK 节点实现腿部 IK

// 在 AnimInstance 中更新脚步 IK 数据
void UMyAnimInstance::UpdateFootIK(float DeltaSeconds)
{
    if (!OwnerCharacter) return;
    
    USkeletalMeshComponent* Mesh = OwnerCharacter->GetMesh();
    UWorld* World = GetWorld();
    
    // 左脚 IK
    UpdateFootTarget(Mesh, World, FName("foot_l"), LeftFootIKOffset, LeftFootNormal);
    // 右脚 IK
    UpdateFootTarget(Mesh, World, FName("foot_r"), RightFootIKOffset, RightFootNormal);
    
    // 计算骨盆偏移（平衡两脚高度差）
    float HeightDiff = LeftFootIKOffset.Z - RightFootIKOffset.Z;
    PelvisOffset = FMath::FInterpTo(
        PelvisOffset, 
        FMath::Min(LeftFootIKOffset.Z, RightFootIKOffset.Z) * 0.5f, 
        DeltaSeconds, 
        10.0f
    );
}

void UMyAnimInstance::UpdateFootTarget(
    USkeletalMeshComponent* Mesh, 
    UWorld* World, 
    FName FootBoneName, 
    FVector& OutOffset, 
    FVector& OutNormal)
{
    // 获取脚骨骼的世界位置
    FVector FootLocation = Mesh->GetSocketLocation(FootBoneName);
    
    // 射线检测（从脚上方往下）
    FHitResult HitResult;
    FVector TraceStart = FootLocation + FVector(0, 0, 100.0f);
    FVector TraceEnd = FootLocation - FVector(0, 0, 100.0f);
    
    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(OwnerCharacter);
    
    if (World->LineTraceSingleByChannel(
        HitResult, TraceStart, TraceEnd, 
        ECC_Visibility, QueryParams))
    {
        // 命中地面
        OutOffset = HitResult.ImpactPoint - FootLocation;
        OutNormal = HitResult.ImpactNormal;
    }
    else
    {
        // 没有地面，恢复
        OutOffset = FVector::ZeroVector;
        OutNormal = FVector::UpVector;
    }
}
```

```cpp
// 完整的布娃娃切换实现
void AMyCharacter::EnableRagdoll()
{
    USkeletalMeshComponent* Mesh = GetMesh();
    
    // 禁用胶囊体碰撞
    GetCapsuleComponent()->SetCollisionEnabled(ECollisionEnabled::NoCollision);
    
    // 停止 CharacterMovement
    GetCharacterMovement()->DisableMovement();
    GetCharacterMovement()->StopMovementImmediately();
    
    // 开启物理模拟（所有骨骼进入物理状态）
    Mesh->SetSimulatePhysics(true);
    Mesh->SetCollisionEnabled(ECollisionEnabled::QueryAndPhysics);
    
    // 应用死亡前的速度，使布娃娃有初始动量
    FVector DeathVelocity = GetVelocity();
    Mesh->AddImpulse(DeathVelocity * Mesh->GetMass() * 0.5f, NAME_None, false);
    
    bIsRagdoll = true;
}

void AMyCharacter::DisableRagdoll()
{
    USkeletalMeshComponent* Mesh = GetMesh();
    
    // 保存当前物理姿势
    Mesh->GetBoneTransform(0); // Root bone world transform
    
    // 关闭物理模拟，恢复动画控制
    Mesh->SetSimulatePhysics(false);
    
    // 如果需要从布娃娃状态"站起来"，需要播放起身蒙太奇
    // ...
    
    bIsRagdoll = false;
}
```

```cpp
UCLASS()
class UMyCharacterAnimInstance : public UAnimInstance
{

public:

    UPROPERTY(BlueprintReadOnly, Category = "Locomotion")
    bool bIsInAir = false;

    bool bIsCrouching = false;

    UPROPERTY(BlueprintReadOnly, Category = "Aiming")
    float AimYaw = 0.0f;

    bool bIsAiming = false;

    UPROPERTY(BlueprintReadOnly, Category = "IK")
    FRotator RightFootIKRotation = FRotator::ZeroRotator;
    
    UPROPERTY(BlueprintReadOnly, Category = "IK")
    float PelvisOffset = 0.0f;

protected:
    virtual void NativeInitializeAnimation() override;
    virtual void NativeUpdateAnimation(float DeltaSeconds) override;
    
private:
    UPROPERTY()
    TObjectPtr<ACharacter> OwnerCharacter;
    
    void UpdateLocomotion(float DeltaSeconds);
    void UpdateAiming(float DeltaSeconds);
    void UpdateFootIK(float DeltaSeconds);
    
    void DoFootTrace(FName FootBoneName, FVector& OutIKTarget, FRotator& OutIKRotation);
};

// MyCharacterAnimInstance.cpp
void UMyCharacterAnimInstance::NativeInitializeAnimation()
{
    Super::NativeInitializeAnimation();
    OwnerCharacter = Cast<ACharacter>(TryGetPawnOwner());
}

void UMyCharacterAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    Super::NativeUpdateAnimation(DeltaSeconds);
    
    if (!OwnerCharacter) return;
    
    UpdateLocomotion(DeltaSeconds);
    UpdateAiming(DeltaSeconds);
    UpdateFootIK(DeltaSeconds);
}

void UMyCharacterAnimInstance::UpdateLocomotion(float DeltaSeconds)
{
    FVector Velocity = OwnerCharacter->GetVelocity();
    Speed = Velocity.Size2D();
    bIsInAir = OwnerCharacter->GetCharacterMovement()->IsFalling();
    bIsCrouching = OwnerCharacter->GetCharacterMovement()->IsCrouching();
    
    // 计算相对移动方向
    if (Speed > 1.0f)
    {
        FRotator ActorRot = OwnerCharacter->GetActorRotation();
        FVector NormalVelocity = Velocity.GetSafeNormal2D();
        float ForwardDot = FVector::DotProduct(NormalVelocity, ActorRot.Vector());
        float RightDot = FVector::DotProduct(NormalVelocity, 
            FRotationMatrix(ActorRot).GetScaledAxis(EAxis::Y));
        Direction = FMath::RadiansToDegrees(FMath::Atan2(RightDot, ForwardDot));
    }
}

void UMyCharacterAnimInstance::UpdateAiming(float DeltaSeconds)
{
    FRotator CharRot = OwnerCharacter->GetActorRotation();
    FRotator AimRot = OwnerCharacter->GetBaseAimRotation();
    FRotator Delta = (AimRot - CharRot).GetNormalized();
    
    // 平滑插值，避免突变
    AimYaw = FMath::FInterpTo(AimYaw, 
        FMath::ClampAngle(Delta.Yaw, -90.f, 90.f), DeltaSeconds, 15.0f);
    AimPitch = FMath::FInterpTo(AimPitch, 
        FMath::ClampAngle(Delta.Pitch, -90.f, 90.f), DeltaSeconds, 15.0f);
}

void UMyCharacterAnimInstance::DoFootTrace(
    FName FootBoneName, FVector& OutIKTarget, FRotator& OutIKRotation)
{
    USkeletalMeshComponent* Mesh = GetSkelMeshComponent();
    if (!Mesh) return;
    
    FVector FootWorld = Mesh->GetSocketLocation(FootBoneName);
    FVector Start = FootWorld + FVector(0, 0, 80.0f);
    FVector End   = FootWorld - FVector(0, 0, 80.0f);
    
    FHitResult Hit;
    FCollisionQueryParams Params;
    Params.AddIgnoredActor(OwnerCharacter);
    
    if (GetWorld()->LineTraceSingleByChannel(Hit, Start, End, ECC_Visibility, Params))
    {
        // 将世界坐标转换为组件坐标系偏移
        FVector ComponentSpaceHit = Mesh->GetComponentTransform()
            .InverseTransformPosition(Hit.ImpactPoint);
        OutIKTarget = ComponentSpaceHit;
        
        // 根据地面法线计算脚部旋转
        FRotator FootRot = FRotationMatrix::MakeFromZX(
            Hit.ImpactNormal, OwnerCharacter->GetActorForwardVector()).Rotator();
        OutIKRotation = FootRot;
    }
    else
    {
        OutIKTarget   = FVector::ZeroVector;
        OutIKRotation = FRotator::ZeroRotator;
    }
}

void UMyCharacterAnimInstance::UpdateFootIK(float DeltaSeconds)
{
    DoFootTrace(FName("foot_l"), LeftFootIKTarget, LeftFootIKRotation);
    DoFootTrace(FName("foot_r"), RightFootIKTarget, RightFootIKRotation);
    
    // 骨盆补偿
    float MinZ = FMath::Min(LeftFootIKTarget.Z, RightFootIKTarget.Z);
    PelvisOffset = FMath::FInterpTo(PelvisOffset, MinZ * 0.5f, DeltaSeconds, 10.0f);
}
```

## 7.4 常见问题解决

### Q1: 动画导入后骨骼旋转错误

**原因**：DCC 软件与 UE 的坐标系不同。

**解决**：
```
Maya 导出设置：
  Up Axis: Y（Maya 默认）→ 在 UE 导入时设置 Transform → Rotation X = -90
  或者在 Maya 中直接以 Z-Up 导出（推荐）

Blender 导出设置：
  Forward: -Y Forward
  Up: Z Up
```

### Q2: 脚步 IK 出现抖动

**原因**：射线检测每帧结果不稳定，或 IK 偏移量变化过快。

**解决**：
```cpp
// 对 IK 目标使用平滑插值
LeftFootIKTarget = FMath::VInterpTo(
    LeftFootIKTarget, NewTarget, DeltaSeconds, 15.0f);
```

### Q3: 动画蒙太奇播放后角色位移消失

**原因**：Montage 没有启用 Root Motion，或 `AnimRootMotionTranslationScale` 被设为 0。

**解决**：
```cpp
// 确认 Montage 中的 AnimSequence 已启用 Root Motion
// 检查 CharacterMovement 设置：
GetCharacterMovement()->AnimRootMotionTranslationScale = 1.0f;
```

### Q4: 多线程 AnimBP 更新崩溃

**原因**：在 `NativeThreadSafeUpdateAnimation` 中访问了非线程安全的对象。

**解决**：
```cpp
// ❌ 错误（Worker Thread 无法安全访问 Actor）
void UMyAnimInstance::NativeThreadSafeUpdateAnimation(float Delta)
{
    Speed = TryGetPawnOwner()->GetVelocity().Size(); // 崩溃！
}

// ✅ 正确（通过 Property Access 系统或在 NativeUpdateAnimation 中缓存数据）
void UMyAnimInstance::NativeUpdateAnimation(float Delta)
{
    // 在游戏线程缓存数据
    if (APawn* Pawn = TryGetPawnOwner())
        CachedSpeed = Pawn->GetVelocity().Size();
}

void UMyAnimInstance::NativeThreadSafeUpdateAnimation(float Delta)
{
    Speed = CachedSpeed; // 安全！读取已缓存的值
}
```

### Q5: Blend Space 动画之间有明显的滑步

**原因**：不同速度的动画步频不匹配（如 Walk 2步/秒，Run 3步/秒，混合时步幅不协调）。

**解决**：
```
方案1：Sync Group（同步组）
  在 Blend Space 中启用 Sync Group，让所有动画的步频强制同步

方案2：Stride Warping（步幅扭曲，UE5）
  使用 Stride Warping AnimGraph 节点动态调整步幅适配速度

方案3：手动对齐帧
  在 DCC 中确保所有运动动画的关键帧（左脚触地、右脚触地）在 Blend Space 中对齐
```

### Pose Search（姿势搜索 / Motion Matching）

Motion Matching 是下一代动画技术，通过从大型动画数据库中实时搜索最匹配的姿势来驱动角色：

```
Motion Matching vs 传统状态机：

传统状态机：
  设计师手动定义状态、转换、混合 → 组合爆炸问题

Motion Matching：
  输入：当前角色姿势 + 期望的未来轨迹
  输出：从动画数据库中搜索最优匹配帧
  优势：过渡自然、维护成本低、支持大量动画数据
  
UE5 插件：Pose Search Plugin（实验性，UE 5.3+）
```

## C. 代码示例库

### 常用工具函数

```cpp
// AnimationUtilities.h - 动画工具函数集合

/** 播放蒙太奇并绑定完成回调 */
static float PlayMontageWithCallback(
    UAnimInstance* AnimInst,
    UAnimMontage* Montage,
    TFunction<void(UAnimMontage*, bool)> OnEnded,
    float PlayRate = 1.0f)
{
    if (!AnimInst || !Montage) return 0.0f;
    
    float Duration = AnimInst->Montage_Play(Montage, PlayRate);
    
    if (Duration > 0.0f)
    {
        FOnMontageEnded EndedDelegate;
        EndedDelegate.BindLambda(OnEnded);
        AnimInst->Montage_SetEndDelegate(EndedDelegate, Montage);
    }
    
    return Duration;
}

/** 安全获取 AnimInstance（带类型检查）*/
template<typename T>
static T* GetAnimInstanceSafe(USkeletalMeshComponent* MeshComp)
{
    if (!MeshComp) return nullptr;
    return Cast<T>(MeshComp->GetAnimInstance());
}

/** 检查蒙太奇是否正在播放指定 Section */
static bool IsMontagePlayingSection(
    UAnimInstance* AnimInst, 
    UAnimMontage* Montage, 
    FName SectionName)
{
    if (!AnimInst || !Montage) return false;
    if (!AnimInst->Montage_IsPlaying(Montage)) return false;
    
    FName CurrentSection = AnimInst->Montage_GetCurrentSection(Montage);
    return CurrentSection == SectionName;
}

/** 获取骨骼的世界变换（线程安全版本）*/
static FTransform GetBoneWorldTransformSafe(
    USkeletalMeshComponent* MeshComp, 
    FName BoneName)
{
    if (!MeshComp) return FTransform::Identity;
    return MeshComp->GetBoneTransform(
        MeshComp->GetBoneIndex(BoneName));
}
```
