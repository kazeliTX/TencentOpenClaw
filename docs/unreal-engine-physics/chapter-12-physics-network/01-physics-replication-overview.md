# 12.1 物理网络同步总览

> **难度**：⭐⭐⭐⭐⭐

## 一、核心挑战

```
物理同步的根本问题：
  物理模拟是确定性的（给定相同输入/状态 → 相同结果）
  但网络延迟导致各端输入时序不同 → 状态发散

  问题分类：
  1. 确定性问题：
     浮点精度差异（不同平台/编译器 → 结果微小不同）
     → 累积误差 → 最终结果天差地别
     
  2. 延迟问题：
     客户端 A 踢了一个箱子 → 服务端 50ms 后收到 → 位置不同
     
  3. 竞态问题：
     两个玩家同时推同一个物体 → 各自算的结果冲突
```

## 二、UE 物理同步架构

```
服务端权威（Server Authority）：
  物理只在服务端真实计算
  客户端显示服务端同步过来的位置（纯接收）
  优点：绝对正确，无作弊空间
  缺点：延迟感（客户端看到的总是"过去的"物体）
  
客户端预测（Client Prediction）：
  客户端在本地先预测物理结果（立即响应）
  服务端计算权威结果并定期发送给客户端
  若差异超过阈值 → 客户端回滚到服务端状态（Rewind）
  优点：手感好（即时反馈）
  缺点：实现复杂，需要处理预测错误
  
混合策略（UE 实际使用）：
  不重要物体（远处箱子）：服务端权威 + 位置插值平滑
  重要物体（玩家控制车辆/角色）：客户端预测 + 服务端校正
```

## 三、UE5 物理网络新特性

```
UE5.1+ 引入：
  Network Physics Prediction（物理预测框架）
  Physics Rewind（物理回卷调试工具）
  Async Physics（异步物理，独立线程）
  
  这些特性让物理网络同步的实现标准化
  → 无需每个项目自己从头造轮子
```
