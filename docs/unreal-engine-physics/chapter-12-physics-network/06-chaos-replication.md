# 12.6 Chaos 物理复制

> **难度**：⭐⭐⭐⭐⭐

```
Chaos 物理复制特点（UE5）：

GeometryCollection（破碎体）复制：
  GC 的每个碎片状态需要同步
  碎片数量可能很多（16~500+）→ 带宽压力大
  
  GC 复制策略：
    方案A：只同步"破碎事件"
      服务端检测到碎片断开 → RPC 通知客户端
      客户端重现相同的破碎（但物理结果可能略有不同）
      带宽极低，同步性中等
      
    方案B：同步 Field System 操作
      服务端应用 Field → Multicast RPC 广播 Field 参数
      客户端重现相同 Field → 物理结果接近但不完全相同
      适合爆炸等较大范围破碎
      
    方案C：权威端同步所有碎片状态（不推荐）
      带宽极大，延迟高，一般不可行

Chaos Cloth 网络同步：
  布料通常不同步（每个玩家在本地各自模拟）
  主玩家的布料状态不同步给其他客户端
  视觉一致性靠：相同的 Cloth 参数 + 近似同步
  
ChaosVehicle 复制（见 Ch12.8）：
  车辆的物理状态（位置/速度/车轮角度）需要高频同步
  UE5 ChaosVehicle 自带预测框架
```
